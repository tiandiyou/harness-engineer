## 二、需求管理阶段

### 2.1 需求来源与分类

```
需求来源
├── 用户反馈 (User Feedback)
├── 业务方需求 (Business Requirement)
├── 技术演进 (Technical Evolution)
├── 竞品分析 (Competitor Analysis)
└── 合规要求 (Compliance)
```

**需求分类**：
| 类型 | 说明 | 优先级 |
|------|------|--------|
| P0 | 阻断性问题，紧急修复 | 立即处理 |
| P1 | 核心功能缺陷 | 本周修复 |
| P2 | 常规需求 | 本Sprint |
| P3 | 优化建议 | 下个Sprint |

### 2.2 需求文档规范

**需求模板 (PRD)**：

```markdown
# 需求文档 ID: REQ-2026-0407-001

## 1. 需求概述
- 需求名称: 用户登录优化
- 需求来源: 产品需求
- 需求描述: 优化登录流程，增加短信验证码登录

## 2. 业务价值
- 提升用户体验
- 降低登录失败率

## 3. 详细需求
### 3.1 功能需求
- FR-01: 用户可通过手机号+短信验证码登录
- FR-02: 验证码有效期5分钟
- FR-03: 验证码输入错误3次后锁定

### 3.2 非功能需求
- NFR-01: 登录响应时间 < 2秒
- NFR-02: 支持高并发 (1000 TPS)

## 4. 验收标准
- [ ] 短信验证码登录功能正常
- [ ] 验证码有效期验证通过
- [ ] 错误3次后锁定功能正常
- [ ] 性能测试通过

## 5. 关联信息
- 关联需求: REQ-2026-0406-005
- 设计文档: DES-2026-0407-001
- 测试用例: TC-2026-0407-001
```

```yaml
## 基本信息
```

```
                      develop
                    ↗      ↖

       ↗         ↖       ↗       ↖
```

### 7.2 Jenkins Pipeline

```groovy
pipeline {
    agent any
    
    environment {
        DOCKER_REGISTRY = 'harbor.huawei.com'
        APP_NAME = 'huawei-sdlo-demo'
        VERSION = "${env.BUILD_NUMBER}"
    }
    
    stages {
        stage('Checkout') {
            steps {
                checkout scm
            }
        }
        
        stage('Build') {
            steps {
                sh 'mvn clean package -DskipTests'
            }
        }
        
        stage('Unit Test') {
            steps {
                sh 'mvn test'
                junit 'target/surefire-reports/*.xml'
            }
        }
        
        stage('Code Scan') {
            steps {
                sh 'mvn sonar:sonar'
            }
        }
        
        stage('Build Image') {
            steps {
                sh """
                    docker build -t ${APP_NAME}:${VERSION} .
                    docker tag ${APP_NAME}:${VERSION} ${APP_NAME}:latest
                """
            }
        }
        
        stage('Deploy to Test') {
            steps {
                sh """
                    kubectl set image deployment/${APP_NAME} \
                    ${APP_NAME}=${APP_NAME}:${VERSION} \
                    -n test
                """
            }
        }
        
        stage('E2E Test') {
            steps {
                sh 'mvn test -Denv=test -Dcucumber.tags="@smoke"'
            }
        }
        
        stage('Deploy to Prod') {
            when {
                branch 'main'
            }
            steps {
                input message: '确认发布到生产环境?', ok: '确认发布'
                sh """
                    kubectl set image deployment/${APP_NAME} \
                    ${APP_NAME}=${APP_NAME}:${VERSION} \
                    -n production
                """
            }
        }
    }
    
    post {
        always {
            cleanWs()
        }
        success {
            echo '构建成功!'
        }
        failure {
            echo '构建失败!'
        }
    }
}
```
